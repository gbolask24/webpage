# Scrape YouTube Video

Scrape a YouTube video's metadata and transcript, then save as a clean markdown file in `outputs/`.

## Variables

video_url: $ARGUMENTS (YouTube video URL, e.g., `https://youtu.be/JQQhT0edXXw`)

---

## Instructions

### Step 1: Read API Key

Read the `.env` file to get the `APIFY_KEY`.

### Step 2: Run Both Scrapers in Parallel

**Transcript scraper** — actor `pintostudio/youtube-transcript-scraper` (ID `faVsWy9VTSNVIhWpR`):

```bash
curl -s "https://api.apify.com/v2/acts/faVsWy9VTSNVIhWpR/runs?token=$APIFY_KEY" \
  -X POST \
  -d '{"videoUrl":"VIDEO_URL","targetLanguage":"en"}' \
  -H 'Content-Type: application/json'
```

**Metadata scraper** — actor `streamers/youtube-scraper` (ID `streamers~youtube-scraper`):

```bash
curl -s "https://api.apify.com/v2/acts/streamers~youtube-scraper/runs?token=$APIFY_KEY" \
  -X POST \
  -d '{"downloadSubtitles":false,"hasCC":false,"hasLocation":false,"hasSubtitles":false,"is360":false,"is3D":false,"is4K":false,"isBought":false,"isHD":false,"isHDR":false,"isLive":false,"isVR180":false,"maxResultStreams":0,"maxResults":1,"maxResultsShorts":0,"preferAutoGeneratedSubtitles":false,"saveSubsToKVS":false,"startUrls":[{"url":"VIDEO_URL"}]}' \
  -H 'Content-Type: application/json'
```

Replace `VIDEO_URL` with the actual URL. Both return a run ID and dataset ID in `data.id` and `data.defaultDatasetId`.

### Step 3: Poll Until Both Runs Succeed

Poll each run status at:
```
https://api.apify.com/v2/actor-runs/{runId}?token=$APIFY_KEY
```

Use an until-loop with `sleep 5` intervals. Check `data.status` for `SUCCEEDED`, `FAILED`, or `ABORTED`.

### Step 4: Fetch Results

Fetch dataset items from:
```
https://api.apify.com/v2/datasets/{datasetId}/items?token=$APIFY_KEY
```

- **Transcript:** Array at `[0].data`, each item has `text`, `start`, `dur`. Join all `text` fields.
- **Metadata:** Single object at `[0]` with `title`, `viewCount`, `likes`, `commentsCount`, `duration`, `date`, `channelName`, `numberOfSubscribers`, `text` (description), `descriptionLinks`, `thumbnailUrl`, etc.

### Step 5: Build the Markdown File

Structure the output as:

```markdown
# {title}

**URL:** {url}
**Thumbnail:** {thumbnailUrl}

| Field | Value |
|---|---|
| Published | {date formatted} |
| Duration | {duration} |
| Views | {viewCount} |
| Likes | {likes} |
| Comments | {commentsCount} |
| Channel | {channelName} (@{channelUsername}) |
| Subscribers | {numberOfSubscribers} |

## Description

{description text}

## Timestamps

{extracted from description if present}

## Links

{extracted from descriptionLinks}

---

## Transcript

{cleaned transcript text, organized into logical paragraphs, section headers where topic shifts are clear}
```

**Transcript cleanup rules:**
- Remove `[music]`, `[applause]`, and similar tags
- Merge overlapping segments into flowing text
- Break into paragraphs every 3-5 sentences at natural topic shifts
- Add `###` section headers when the video has clear parts/chapters
- Preserve ALL content including CTAs, sponsor reads, and outros — do not trim anything
- Fix obvious speech-to-text errors (e.g., "CloudCode" → "Claude Code", "aache" → "Apache")

### Step 6: Save the File

Save to `outputs/video-{slug}.md` where `{slug}` is a kebab-case version of the video title (short, descriptive, max ~5 words).

### Step 7: Report

Tell the user:
- File path saved
- Video title, views, likes, duration
- Confirm transcript length (segment count and approximate word count)
